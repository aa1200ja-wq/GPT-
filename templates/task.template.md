- 直接使用 Codex Cloud 平台提供的工作分支
- 不得因本地沒有基準分支而停止
- 不得因不存在 `origin` remote 而停止
- 不得執行 `git fetch`
- 不得手動建立或切換分支
- 不得自行推送或合併至基準分支
- 任務完成後保留平台產生的變更
- Pull Request 由 Codex Cloud 介面建立
- 若尚未建立 Pull Request，於回報中填寫「待 Codex Cloud 介面建立」

1. 執行 `git diff --check`

不適用的測試指令必須刪除，不得保留未替換的佔位文字。

## 九、交付要求

完成任務後必須：

1. 更新 `handoff/report.md`
2. 填寫任務編號與任務名稱
3. 說明實際完成內容
4. 列出所有修改、新增、刪除、移動或重新命名的檔案
5. 寫明全部測試指令與結果
6. 填寫未完成項目
7. 填寫問題與風險
8. 填寫 Codex Cloud 工作分支資訊
9. Pull Request 尚未建立時，填寫「待 Codex Cloud 介面建立」
10. 保留平台產生的變更
11. 完成回報後立即停止

## 十、停止條件

遇到以下任一情況，必須立即停止並回報：

- 任務狀態不是「已批准執行」
- 無法讀取 `AGENTS.md`
- 無法讀取本任務
- 需求不清楚或資料不足
- 需要修改未授權檔案
- 任務要求操作其他儲存庫
- 測試無法執行
- 發現可能破壞既有功能
- 需要新增套件但任務未授權
- 發現密碼、API Key、Token 或其他機密資料
- 無法依照允許範圍完成任務

未經 GPT／小安驗收及老闆說「通過」，不得自行執行下一個任務。
