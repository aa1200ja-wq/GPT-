# Codex 專案執行規則

## 一、允許操作範圍

- 唯一允許操作的儲存庫：`aa1200ja-wq/GPT-`
- 禁止存取、修改或提交至：`aa1200ja-wq/online_rollcall`
- 預設分支：`main`
- 不得直接修改 `main`
- 每次任務必須使用獨立分支，完成後建立 Pull Request

## 二、開始任務前

1. 先閱讀 `handoff/task.md`
2. 確認任務狀態為「已批准執行」
3. 任務狀態不是「已批准執行」時，立即停止，不得修改任何檔案
4. 只執行 `handoff/task.md` 內明確列出的唯一任務
5. 需求不清楚或資料不足時，立即停止並回報，不得自行猜測

## 三、執行限制

- 一次只處理一個原子任務
- 不得擴充任務範圍
- 不得順便重構無關程式
- 不得任意新增套件
- 不得任意刪除、移動或重新命名檔案
- 不得破壞既有功能
- 不得寫入密碼、API Key、Token 或其他機密資料
- 修改前先閱讀相關檔案
- 修改後必須執行任務指定的測試
- 無法執行測試時，必須明確說明原因

## 四、完成任務後

完成目前任務後，必須更新 `handoff/report.md`，至少填寫：

- 執行狀態
- 完成內容
- 修改檔案
- 測試結果
- 未完成項目
- 問題與風險
- Pull Request 資訊

完成回報後立即停止。

未經 GPT 驗收及老闆說「通過」，不得執行下一個任務。
