# Codex 專案執行規則

## 一、允許操作範圍

* 唯一允許操作的儲存庫：`aa1200ja-wq/GPT-`
* 禁止存取、修改或提交至：`aa1200ja-wq/online_rollcall`
* Codex Cloud 任務以介面選定的基準分支為來源，目前預設為 `main`
* 不得直接推送或合併至 `main`
* Codex Cloud 必須使用平台提供的工作分支
* 不得自行執行 `git fetch origin/main`
* 不得自行建立、切換或重新命名分支
* 任務完成後先保留平台產生的變更並停止
* Pull Request 由 Codex Cloud 介面建立

## 二、開始任務前

1. 先完整閱讀 `handoff/task.md`
2. 確認任務狀態為「已批准執行」
3. 任務狀態不是「已批准執行」時，立即停止，不得修改任何檔案
4. 只執行 `handoff/task.md` 內明確列出的唯一任務
5. 確認允許修改的檔案範圍
6. 確認禁止修改的檔案範圍
7. 需求不清楚或資料不足時，立即停止並回報，不得自行猜測

## 三、執行限制

* 一次只處理一個原子任務
* 不得擴充任務範圍
* 不得順便處理下一個任務
* 不得重構無關程式
* 不得任意新增套件
* 不得任意刪除檔案
* 不得任意移動檔案
* 不得任意重新命名檔案
* 不得修改未授權檔案
* 不得破壞既有功能
* 不得寫入密碼、API Key、Token 或其他機密資料
* 修改前必須先閱讀相關檔案
* 修改後必須執行任務指定的測試
* 無法執行測試時，必須明確說明原因
* 發現可能影響其他儲存庫時，必須立即停止
* 禁止操作 `aa1200ja-wq/online_rollcall`

## 四、Codex Cloud 工作環境規則

* Codex Cloud 可能只提供平台工作分支，例如 `work`
* 不得因本地沒有 `main` 分支而停止
* 不得要求存在 `origin` remote
* 不得要求執行 `git fetch`
* 不得要求手動推送遠端分支
* 應直接在平台提供的工作分支完成任務
* 完成修改後保留變更，交由 Codex Cloud 介面建立 Pull Request
* 不得自行合併 Pull Request
* 不得直接修改或推送至 `main`

## 五、完成任務後

完成目前任務後，必須更新 `handoff/report.md`，至少填寫：

* 任務編號
* 任務名稱
* 執行狀態
* 完成內容
* 修改檔案
* 新增檔案
* 刪除或重新命名檔案
* 測試指令
* 測試結果
* 未完成項目
* 問題與風險
* 需要 GPT／老闆確認的事項
* Codex Cloud 工作分支資訊
* Pull Request 資訊；若平台尚未建立，填寫「待 Codex Cloud 介面建立」

## 六、完成後停止規則

完成回報後必須立即停止。

不得：

* 自行修改 `handoff/task.md`
* 自行更改任務批准狀態
* 自行建立下一個任務
* 自行執行下一個任務
* 自行合併 Pull Request
* 自行修改 `main`
* 自行操作其他儲存庫

未經 GPT／小安驗收及老闆說「通過」，不得執行下一個任務。
